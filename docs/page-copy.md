# Page Copy

本文档用于沉淀 V1 的前端页面文案，供产品、设计和前端统一使用。

## 1. Employee H5

### 1.1 匿名提问页

- Page Title: `匿名提问`
- Subtitle: `你可以匿名提交问题，并选择希望的处理方式。请尽量不要填写可能识别个人身份的细节。`

#### 处理方式区

- `公开讨论`
  - `提交后进入公开问题池，其他同事可看到。适合希望提前公开讨论的共性问题。`
- `会上公开（默认）`
  - `会前不公开，大会现场统一展示并答复。适合大多数希望在会上被回应的问题。`

#### 输入区

- Label: `请输入你的问题`
- Placeholder: `请尽量直接描述问题本身，避免填写姓名、团队、项目名、具体时间等可能识别身份的信息。`
- Character Counter Format: `32 / 500`

#### 提示区

- `系统默认不收集姓名、工号等身份信息。`
- `原始问题不会被改写。`
- `如涉及举报、骚扰、歧视、合规或个人权益事项，请走正式渠道。`

#### Button

- Primary Button: `匿名提交`

#### Validation Messages

- `请输入问题内容`
- `问题至少 10 个字`
- `问题请控制在 500 字以内`

### 1.2 提交前确认弹窗

#### 公开讨论

- Title: `确认公开讨论？`
- Body: `你的问题会在审核后进入公开问题池，其他同事可以看到原文。`
- Primary Button: `确认提交`
- Secondary Button: `取消`

#### 会上公开

- Title: `确认会上公开？`
- Body: `你的问题会在大会前保留不公开，并视安排在会上展示。`
- Primary Button: `确认提交`
- Secondary Button: `取消`

### 1.3 提交成功页

#### 公开讨论

- Title: `提交成功`
- Body: `你的问题将在审核后进入公开问题池。`

#### 会上公开

- Title: `提交成功`
- Body: `你的问题会在大会前保留不公开，并视安排在会上展示。`

#### Common Actions

- Primary Button: `继续提问`
- Secondary Button: `查看公开讨论`

Note:

- `查看公开讨论` 仅在 `公开讨论` 和部分默认成功页场景中展示。

### 1.4 公开问题池

- Page Title: `公开讨论`
- Intro: `这里只展示选择“公开讨论”且审核通过的问题。`
- Search Placeholder: `搜索公开问题`
- Empty State: `暂时还没有公开问题`
- Empty State Action: `去提问`

#### 卡片字段

- 问题原文
- 所属议题
- 答复状态：`待回应 / 已回应 / 会后补答`
- 日期：只显示到天，例如 `2026-03-23`

## 3. Shared Risk Copy

以下风险提示建议在员工端页面固定保留：

- `系统默认不收集姓名、工号等身份信息。`
- `请避免填写具体人名、项目名、时间点等可识别信息。`
- `原始问题不会被改写，但不代表所有原文都会被公开展示。`
- `如涉及举报、骚扰、歧视、舞弊、合规或个人权益事项，请走正式渠道。`
